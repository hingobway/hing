The Discord Manifesto

By Michael Foster

A document detailing various ways to improve the runnings of the group Discord chat, timeline, reasons for the changes, etc.

Section 1: Rankings

Our current ranking system provides some customization of rankings– a 3 level system like our own runs well for the most part, but sometimes you need a few more roles so more people can have some kind of privilege. Because there are people in the Discord who deserve to have some control, but leveling them up to Mod rank is just too much. Also, if someone has done something that warrants punishment, it is extremely difficult to punish them because of the fact that @everyone rights are very difficult to revoke, even from specific people, and every right must be removed manually.

So what’s the solution? Here is the ranking system I suggest:

Rank Suggestions
Each rank has all the rights of all the level below it as well.

  1. @everyone - absolutely no permissions
  2. @Time Out - *a level with minimal permissions. negotiable, but I’m thinking the following:
    • read messages
    • join #general
  3. @normal - the average user. what @everyone is right now.
  4. @Mod1 - a very low level mod with some basic nominal rights.
    • manage emojis
    • send TTS
    • mention @everyone
    • create instant invites
  5. @Mod2 - slightly higher level moderator with some voice management rights.
    • server mute users
    • server deafen users
    • move users to different voice channels
  6. @Manager1 - control over chat and a few other permissions.
    • pin messages
    • delete messages
    • manage nicknames
    • manage webhooks (don’t think this will mean much to most people but it will to a few)
    • kick users
  7. @Manager2 - starts adding control over server settings.
    • add/edit/delete chatrooms
    • add/edit/delete voice channels
    • ban users
  8. @Admin - every control except editing their own role
  9. @owner - not a real role, but owner has control over actually everything
    Add your suggestions!

Edits and suggestions are welcome, but the key point here is that 9 ranks creates so much more customization than the 3-4 we have right now, and will give us a chance to give many more people some kind of role and some kind of control, as well as make sure it is as easy as possible to change people’s role in the group.

Section 2: Bots and Welcoming Procedures

Let’s put it this way– it’s a bit of a free-for-all right now in terms of discord bots. People post bot commands in all chatrooms, people spam bot commands and mess with songs– it’s not very conducive to a solid and well-controlled group chat. So there are a few things I think we should do regarding this system.

Bot Restrictions

  • All bot commands should be restricted to the #cmds chatroom (see chatrooms for more on this)
  • Bot permission levels should be given out, not just taken by high-level admins
    • possibly associated with a @manager1 status, but it’s debatable
  • A main management bot must be chosen. Conditions may include:
    • close to 100% uptime
    • autorole capabilities
    • custom command capabilities
    • chatroom restrictions
    • Add your suggestions!

Welcoming Procedures are currently completely obsolete– it’s one of the features of Discord we truly haven’t explored at all. Some of the things involve the ranking system, but there are some other options as well.

Welcoming Procedures

  • On joining, automatically assign the @normal role to the user.
  • Various messages should be triggered on entering:
    • General message in chat announcing a new user
    • DM to the user highlighting:
    • rules
      • welcome message
      • inner workings, etc
      • Add yours!

Section 3: Chatrooms

The three chatrooms we have currently are usually more or less fine– normal chatting happens in #general, bot commands go in #music, and tech help goes in #tech-sup. However, there are some issues with this system. Often people don’t follow the rules of what goes where, or sometimes it just isn’t clear. That being said, here are the changes I think should be made.

Chatroom Fixes

  • Impose strict deletion rules on content that isn’t where it belongs– anything that is in the wrong chat should be deleted
  • Impose punishments on users who repeatedly and purposefully misuse chatrooms
  • Not all chatrooms have to let everyone talk. In other words, Instead of relying on pinned messages, you could have chatrooms as a “rules” page, a “messageboard”, etc.

All of that being said, here are the chatrooms that I suggest.

List of Chatrooms

  • #welcome - the place where you start. no talking aloud, from anyone
    • anyone can be here
    • no one can chat here
    • #general could become this– this is because #general has certain limitations on it that a new one would not. Don’t worry, message history will stay if people wish to see it, but it will be mainly just for new users.
  • #rules - the place to see general rules for the group. no chatting aloud(*)
    • anyone Manager2+ can post here
    • posting should be pretty sporadic, only if new rules are decided on
    • anyone can see this chat, no #matter what
  • #main - the replacement for #general
    • @normal+ can chat here, post whatever here
    • main chatroom to chat however people like
    • no bots should function here
  • #bot - a chat for posting bot commands
    • everyone should have this channel muted
    • all bot interactions should be limited to here.
  • #highclass - chatroom for “high-class citizens” of the server
    • anyone @mod1+ can chat here.
    • a hangout for the slightly cool people on the server
  • #midmods - chatroom for relatively top-notch citizens
    • anyone @manager1+ can chat here.
    • a hangout for the pretty cool people on the server
  • #admin - chatroom for the true OGs
    • only @admin+ can go here
    • we talkin white-tie exclusive
  • Add yours!
